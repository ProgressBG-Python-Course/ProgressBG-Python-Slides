What encoding means?

What encoding means?

To store anything in a computer, you must first encode it, i.e. convert it to bytes. For example:
If you want to store music, you must first encode it using MP3, WAV, etc.
If you want to store a picture, you must first encode it using PNG, JPEG, etc.
If you want to store text, you must first encode it using ASCII, UTF-8, etc.

Unicode Encoding

A Unicode string is a sequence of code points, which are numbers from 0 through 0x10FFFF (1,114,111 decimal)
This sequence needs to be represented as a set of bytes (1 byte can store values from 0 to (2**8)-1) in memory).
The rules for translating a Unicode string into a sequence of bytes are called an encoding
There are various unicode encodings ( UTF-8, UTF-16, UTF-32 and other), but most widely used is UTF-8 which is a variable width character encoding capable of encoding all valid code points in Unicode using one to four bytes

The bytes object

The bytes object

Bytes Object

A bytes object is a built-in data type in Python representing a sequence of bytes.
Bytes objects are created using the bytes() constructor or by using a bytes literal prefixed with b (bytes string).
Bytes objects are immutable (cannot be changed after creation).
They are typically used to store raw binary data, such as images, audio files, or network packets.
Example:

                # Define a byte string
                byte_string = b'Hello'
                print("Byte String:", byte_string)

                # Create a bytes object with the same bytes as in byte string above
                byte_data = bytes([72, 101, 108, 108, 111])
                print("Bytes Object:", byte_data)

                # Check if both objects contain the same bytes
                if byte_string == byte_data:
                    print("The byte string and the bytes object contain the same sequence of bytes.")
                else:
                    print("The byte string and the bytes object do not contain the same sequence of bytes.")


                # Byte String: b'Hello'
                # Bytes Object: b'Hello'
                # The byte string and the bytes object contain the same sequence of bytes.
More on bytes objects: Bytes objects and Bytearrays

Base64

Base64

Overview

Base64 is a binary-to-text encoding scheme that represents binary data in an ASCII string format.
It is commonly used to encode binary data, such as images, audio files, or binary protocol messages, into a format that is suitable for transmission over text-based channels, such as email or HTTP.
Base64 encoding is not a form of encryption or compression; it is simply a way to represent binary data using printable ASCII characters. Base64-encoded data can be easily decoded back to its original binary form.
In Base64 encoding, each group of three bytes of input data is mapped to four ASCII characters. The resulting Base64-encoded string may contain alphanumeric characters, plus two additional characters (usually + and /) for encoding, along with an optional padding character (=) at the end to ensure that the length of the encoded string is a multiple of four.

base64: encode-decode in Python

Python's base64 module provides:
encoding bytes-like objects to ASCII bytes (base64.b64encode() )
decoding bytes-like objects or strings containing ASCII to bytes (base64.b64decode() )
Example

                import base64

                # base64 needs binary data:
                data_bytes = b'abracadabra'

                # encoding binary data to Base64
                data_base64 = base64.b64encode(data_bytes)
                print(f'data_base64: {data_base64}')

                # decoding Base64-encoded data back to binary
                decoded_data_bytes = base64.b64decode(data_base64)
                print("Decoded:", decoded_data_bytes)

                # data_base64: b'YWJyYWNhZGFicmE='
                # Decoded: b'abracadabra'

Popular use cases for encoding data in Base64

Email Attachments: Email systems typically transmit messages using plain text. When attaching binary files (such as images or documents) to an email, these files are Base64 encoded to ensure they can be safely transmitted as part of the email message.
HTTP Transfers: While HTTP is primarily a text-based protocol, it supports the transmission of binary data using techniques like MIME encoding. Base64 encoding is often used to represent binary data within HTTP messages, such as when uploading files via HTML forms or when transferring binary data in API requests and responses.
Data URI Schemes: Data URIs allow resources to be embedded directly into HTML or CSS documents. When embedding binary data (such as images or fonts) using data URIs, the data is typically Base64 encoded to ensure compatibility and portability across different browsers and platforms.
Authentication Credentials: Base64 encoding is commonly used to encode authentication credentials (such as usernames and passwords) for HTTP Basic Authentication. While Base64 encoding does not provide encryption, it is used to represent the credentials as a text string that can be included in HTTP headers.
Serialization Formats: Base64 encoding is often used in serialization formats (such as JSON or XML) to represent binary data as a string value. This allows binary data to be included in structured data formats without requiring special handling or escaping.
