Code Productivity: Linting and Formating

Overview of Linting

What and Why?

Linting is the process of running a program that will analyse code for potential errors and bad practices.
Code Quality and Maintenance: Linting helps in keeping the code quality high and makes maintenance easier by ensuring consistency and adherence to coding standards.
Early Bug Detection: It assists in identifying potential errors and bugs at an early stage, saving time and resources in the development cycle.
Improved Readability: Enforcing a consistent coding style improves readability and understanding of the code, which is particularly beneficial in team environments.
The most popular linting tools for Python are Pylint and Flake8.
Pylint: A highly configurable tool that checks for errors, enforces a coding standard, and looks for code smells. It can also be customized with plugins.
Flake8: A wrapper around PyFlakes, pycodestyle, and Ned Batchelder’s McCabe script, offering an easy way to check your code against coding style (PEP 8), programming errors, and complexity.

Setting Up Pylint

Pylint: Installation

Install Pylint (Optional if you intend to use it only with VSCode which has extension for it)

                pip install pylint
Running the Linter
Once installed, you can run Pylint on a Python file or project like so:

                pylint myfile.py
                pylint myproject/
Enabling Pylint in VSCode
Install the official "Pylint" extension by Microsoft.
You can customize Pylint's behavior by creating a .pylintrc file in your project's root directory. This file allows you to enable or disable checks, set preferred coding standards, and more.
Reference: Linting Python in Visual Studio Code

Automating Code Formatting with Black

Introduction

What is black?
Black is a highly opinionated code formatter for Python that formats code in a uniform way.
It adheres closely to PEP 8 standards, with some additional rules to ensure code readability and reduce bikeshedding over formatting styles in collaborative projects.
Install Black in your dev environment (Optional if you intend to use it only with VSCode which has extension for it)

                pip install black
Or install the VSCode extension "Black Formatter" by Microsoft
Reference: Formatting Python in VS Code

Using Black with VSCode

Set black as default formatter
you can set it as the default formatter for all Python files by setting "editor.defaultFormatter" in your User settings.json file, under a [python] scope.

                "[python]": {
                  "editor.defaultFormatter": "ms-python.black-formatter"
                }
Format your code
You can format your code by right-clicking on the editor and selecting Format Document, or by using the Ctrl+Shift+I keyboard shortcut.
You can also add the following setting to your User settings.json file to enable formatting on save for your code:

                "[python]": {
                    "editor.formatOnSave": true
                  }

Pylint vs. Black: Which One should I Use?

While both Pylint and Black are valuable tools for Python development, they serve distinct purposes:
Pylint:
Focuses on code analysis and linting, identifying potential errors, stylistic inconsistencies, and best practice violations.
Offers detailed reports and suggestions for improvement.
Highly customizable to various style guides and project needs.
Black:
Focuses on automatic code formatting according to a specific style guide (PEP 8 by default).
Enforces consistent formatting for improved readability and reduced style-related conflicts.
Offers limited customization options.

Project Documentation for Contributors

A README file at the root directory should give general information to both users and maintainers of a project
A LICENSE file should always be present and specify the license under which the software is made available to the public.
A TODO file (or section in README) should list the planned development for the code.
A CHANGELOG file (or section in README) should present a short overview of the changes in the code base for the latest versions.
